C++ 对象模型

在c++中，
有两种class data members:
1.static
2.nostatic

有三种class member functions:
1.static
2.nostatic
3.virtual

How is the class to be representd within the machine that is how do we model the various types of data and fuction members

A Simple Object Model

为了简化编译器而设计，却浪费了空间和效率。
在这种对象模型中,对象的内部结构是一个顺序槽，每个槽指向一个成员。
每个成员按照其声明顺序分配slot。

A Table-driven Object Model

为了统一不同class的object内部结构，所有的对象内部只有两个指针，分别指向Data Member Table和Function Member Table两张表。
其中，Data Table 直接存储数据，Function Table 是一个顺序槽，每一个槽对应一个成员函数。

The C++ Object Model

派生自简单对象模型，
nostatic data members 直接分配给每个object.
static data members 存在单个object之外.
static and nostatic function members 也存在object之外.
virtual functions members:
1一张虚函数表
2.一个指向虚函数表的指针
模型优点:节省空间，运行时效率更高
模型缺点: 当类的nostatic data members被修改需要重新编译程序
在双表单模型提供了较大的弹性，因为它多提供了一层间接性，但因此付出空间和执行效率的代价。
